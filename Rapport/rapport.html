Le projet suit une architecture client-serveur MVC :

  • Backend (Flask) : API REST organisée en Blueprints (auth, profil, annonces). Un context processor global injecte l'utilisateur connecté dans tous les templates sans répétition de code.
  • Frontend (Jinja2 / HTML / Bootstrap) : Système d'héritage de templates centré sur base.html. La sidebar, la navigation et l'avatar utilisateur sont gérés une seule fois.
  • Temps réel (Flask-SocketIO) : Communication bidirectionnelle pour la messagerie instantanée.
  • Base de données (MySQL) : Stockage des utilisateurs, annonces, messages, notifications et matchings.

3. Corrections et Améliorations Techniques

3.1 Problème : jinja2.exceptions.UndefinedError — 'user' is undefined

Lors de l'accès à la route /mes-annonces (et à d'autres routes des blueprints), Flask levait une erreur UndefinedError: 'user' is undefined car le template base.html utilise {{ user.initiales }}, {{ user.nom }} et {{ user.filiere }} dans la sidebar, mais la variable user n'était pas passée au render_template de chaque route.

Solution retenue : Context Processor global

Plutôt que de répéter user=get_current_user() dans chaque appel render_template de tous les blueprints, un context processor a été ajouté une seule fois dans App_final.py :

@app.context_processor
def inject_user():
    _, user_context = get_current_user()
    return dict(user=user_context)

Ce mécanisme Flask injecte automatiquement user dans le contexte de tous les templates, y compris ceux rendus depuis les blueprints (annonces.py, profil.py, auth.py, messagerie.py), sans aucune modification dans ces fichiers.

3.2 Route /mes-annonces manquante

La page mes_annonces.html référençait des routes (/annonces/supprimer/<id>, /annonces/modifier/<id>) qui n'étaient pas définies dans annonces.py. Les routes suivantes ont été ajoutées :

Méthode Route Description
GET /mes-annonces Affiche les annonces de l'utilisateur connecté
POST /annonces/modifier/<id> Modifie une annonce (type, format, statut, matières, dispos, description)
POST /annonces/supprimer/<id> Supprime une annonce et ses liaisons (ANNONCE_MATIERE, ANNONCE_DISPONIBILITE, Réponses)

Chaque route vérifie que l'annonce appartient bien à l'utilisateur connecté avant toute modification ou suppression (contrôle d'ownership).

5. Manuel d'Utilisation

5.1 Messagerie

La page principale de messagerie se compose de deux zones :

  • Panneau gauche : Liste des conversations triées par date, avec le dernier message et le nombre de messages non lus.
  • Panneau droit : Fenêtre de chat affichant l'historique des messages de la conversation sélectionnée.

Fonctionnalités :

  • Consulter une conversation : Cliquer sur une conversation dans la liste pour charger son historique.
  • Envoyer un message : Saisir le texte dans le champ en bas et appuyer sur Entrée ou cliquer sur le bouton ➤.
  • Messages en temps réel : Les nouveaux messages apparaissent instantanément grâce à SocketIO.
  • Marquage automatique : Les messages sont marqués comme lus dès que la conversation est ouverte.
  • Recherche : Filtrer les conversations par nom dans la barre de recherche.
  • Indicateur de non-lus : Un badge bleu sur une conversation indique les messages non lus.

5.2 Annonces

  • Explorer les annonces : La page Annonces affiche les offres et demandes de mentorat des autres étudiants, filtrées automatiquement selon les matières maîtrisées et les lacunes de l'utilisateur.
  • Filtres disponibles : Par matière, format de session (présentiel / en ligne / les deux), disponibilité et type (offre / demande).
  • Répondre à une annonce : Cliquer sur le bouton de réponse, saisir un message d'accompagnement et soumettre. Une réponse par annonce est autorisée.

5.3 Mes Annonces

  • Publier une annonce : Accéder à Mes annonces via la barre latérale, choisir le type (offre/demande), le format, les matières, les disponibilités et une description.
  • Modifier une annonce : Cliquer sur ✏️ Modifier pour déplier le formulaire d'édition inline. Les champs type, format, statut, matières, disponibilités et description sont modifiables.
  • Fermer une annonce : Changer le statut sur "Fermée" dans le formulaire de modification.
  • Supprimer une annonce : Cliquer sur 🗑 Supprimer avec confirmation. Toutes les liaisons (matières, disponibilités, réponses) sont supprimées en cascade.

5.4 Notifications

  • Cliquer sur l'icône 🔔 dans la barre de navigation pour ouvrir le panneau des notifications.
  • Les notifications sont classées par type (message, match, réponse) avec un code couleur.
  • Le badge rouge sur l'icône indique le nombre de notifications non lues, mis à jour automatiquement toutes les 3 secondes.
